# RV32IM Core Glossary

This glossary explains recurring abbreviations, signals, and module names so that every reference in the RTL and documentation is unambiguous.

| Term / Shortcut | Expansion | Description / Where Used |
| --- | --- | --- |
| **RV32I / RV32IM** | RISC-V 32-bit Integer / Integer + Multiply-Divide | Instruction set implemented by the core (`riscv_core.v`, `control_unit.v`). "M" indicates MUL/DIV/REM instructions are supported by `alu.v`. |
| **ISA** | Instruction Set Architecture | Contract describing available instructions; informs `control_unit.v` decode logic. |
| **PC** | Program Counter | 32-bit register that holds the address of the current instruction (`riscv_core.v`). |
| **PC+4 / PC4** | Program Counter plus 4 bytes | Next sequential instruction address; forwarded through pipeline registers (`id_ex`, `ex_mem`, `mem_wb`). |
| **IF/ID** | Instruction Fetch / Instruction Decode | Pipeline register module `if_id.v` storing fetched PC and instruction. |
| **ID/EX** | Instruction Decode / Execute | Pipeline register `id_ex.v` passing decoded operands/control into EX. |
| **EX/MEM** | Execute / Memory | Pipeline register `ex_mem.v` isolating ALU results before memory. |
| **MEM/WB** | Memory / Write-Back | Pipeline register `mem_wb.v` holding data for register write-back. |
| **WB** | Write-Back | Final pipeline stage that writes into the register file (`regfile.v`). |
| **ALU** | Arithmetic Logic Unit | Module `alu.v` computing arithmetic, logic, shifts, and MUL/DIV/REM. Controlled via `alu_ctrl`. |
| **ALU_CTRL** | ALU Control Code | 5-bit signal selecting the ALU operation (`defines.vh`, `control_unit.v`, `alu.v`). |
| **REGFILE** | Register File | Dual asynchronous read, synchronous write bank (`regfile.v`) storing 32 general-purpose registers x0–x31. |
| **RS1 / RS2 / RD** | Register Source 1 / Source 2 / Destination | Field names within RISC-V instructions; appear as ports in decode/pipeline modules. |
| **IMEM / DMEM** | Instruction Memory / Data Memory | Harvard interfaces exposed by `riscv_core.v` and modeled in `tb_riscv_core.v`. |
| **IMEM_EN** | Instruction Memory Enable | Constant-high signal requesting instruction fetch each cycle. |
| **DMEM_WE / DMEM_RE** | Data Memory Write Enable / Read Enable | Control bits for store/load operations leaving the MEM stage. |
| **DMEM_SIZE** | Data Memory Size | 2-bit field encoding byte (`00`), halfword (`01`), or word (`10`) transfers; generated by `control_unit.v` and consumed in MEM stage. |
| **MEM_UNSIGNED** | Unsigned Load Flag | Indicates zero-extension for LB/LH loads; stored in `id_ex` and `ex_mem`. |
| **WB_SEL** | Write-Back Select | 2-bit mux choose between ALU result, load data, or PC+4 for register writes. |
| **IMM_GEN** | Immediate Generator | Module `imm_gen.v` producing sign/zero-extended immediates for I/S/B/U/J formats. |
| **HAZARD UNIT** | Hazard Detection Unit | Module `hazard_unit.v` issuing `stall`, `pc_write`, `if_id_write`, `id_ex_flush`, and forwarding selects. |
| **FORWARD_A / FORWARD_B** | Forwarding Select Lines | 2-bit signals choosing operand sources (0=original, 1=MEM/WB, 2=EX/MEM). |
| **LOAD-USE HAZARD** | Dependency between load result and following instruction | Detected when ID stage consumes the pending load’s destination; handled by stalling PC/IF/ID and flushing ID/EX. |
| **BRANCH_TYPE** | Encoded Branch Comparator Mode | Equals `funct3` for branch opcodes; used in EX to evaluate BEQ/BNE/BLT/etc. |
| **JAL / JALR** | Jump and Link / Jump and Link Register | Control bits from decoder; cause PC redirect and write PC+4 to destination register. |
| **PREDICT NOT TAKEN** | Branch Prediction Strategy | Default assumption that branches fall through; resolved in EX and flushes pipeline on mispredict (documented in `ARCHITECTURE_SOP.md`). |
| **$readmemh** | Verilog system task | Loads `mem/program.hex` into instruction memory inside `tb_riscv_core.v`. |
| **$dumpfile / $dumpvars** | Verilog system tasks | Enable waveform dumping to `cpu_wave.vcd`. |
| **IVERILOG / VVP** | Icarus Verilog Compiler / Runtime | Toolchain commands executed via `make sim` and `make run`. |
| **GTKWAVE** | Waveform Viewer | Used by `make wave` to inspect `cpu_wave.vcd`. |
| **NOP** | No Operation | Encoded as `ADDI x0, x0, 0` (hex `00000013`); used to pad program memory in `mem/gen_program.py`. |
| **Fibonacci Program** | Demo workload | Sequence generated in `mem/gen_program.py` to exercise forwarding, load-use hazards, and branches. |
